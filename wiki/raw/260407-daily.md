# A. Obsidian CLI Cheatsheet

Source: [Obsidian docs](https://help.obsidian.md/cli)

> Obsidian CLI = command line interface điều khiển Obsidian từ terminal. Yêu cầu Obsidian 1.12 installer + đang chạy + bật tại Settings → General → CLI.

## Các lệnh hay dùng

```bash
# Đọc/tạo/sửa note
obsidian read file="My Note"
obsidian create name="New Note" content="# Hello" silent
obsidian append file="My Note" content="New line"

# Search & navigate
obsidian search query="keyword" limit=10
obsidian backlinks file="My Note"
obsidian tags sort=count counts

# Daily note
obsidian daily:read
obsidian daily:append content="- [ ] Task mới"

# Properties (frontmatter)
obsidian property:set name="status" value="done" file="My Note"
```

**Mẹo:** `--copy` = clipboard, `silent` = không mở file, `total` = đếm.

## Cú pháp tham số

| Dạng | Ví dụ |
|---|---|
| Parameter | `name="My Note"` |
| Flag | `silent`, `overwrite` |
| File target | `file=<name>` (resolve như wikilink) |
| Path target | `path=folder/note.md` |
| Vault target | `vault="My Vault"` |

## Ứng dụng cho vault này

- Gắn vào `automation/` daemon: `obsidian create` thay viết file trực tiếp
- Thay `daily_briefing_generator.py` bằng `obsidian daily:append`
- Script quick-capture từ terminal vào `quick-notes.md`

# C. HPS Tracing Sync Pipeline

Status: Approved

## Goal

Sync data từ HPS MSSQL (`HPS_TEST_DB` @ 10.43.75.99) vào `integrated_transaction_tracing` (Postgres) → hiển thị tab `Tracing Etd/Eta` trong Data Lake.

## Architecture

```
HPS MSSQL → TransactionTrackingSyncLogic.queryTracings(sourceDb=HPS)
  → groovy QueryTransactionTracings → TracingEvent (DTO)
  → Kafka Producer → Consumer → integrated_transaction_tracing (Postgres)
```

## Components

- `TracingEvent.java` — DTO
- `TransactionTrackingSyncLogic.java` — query + map
- `TracingEventProducer.java` + `TracingEventConsumer.java` — Kafka
- `BFSOneSyncTransactionTrackingCronJob.java` — cron 30min
- `BatchSyncService.syncTransactionTrackings(ctx, sourceDb)` — RPC
- Groovy: `QueryTransactionTrackings` trong `BFSOneSyncSql.groovy`
- Kafka topic + HPS tenant registration

## SQL (giả định, cần probe confirm)

```sql
SELECT t.TransID, td.MAWBNo, t.TpyeofService, t.TransDate, t.LoadingDate, t.ArrivalDate,
  t.TransitETD, t.TransitETA, t.PortofLading, fr.AirPortID, t.PortofUnlading, toLoc.AirPortID,
  td.ShipmentType, td.CustomerName, t.WhoisMaking, ui.FullName, td.OperationPIC, td.SalesmanName
FROM Transactions t
LEFT JOIN TransactionDetails td ON td.TransID = t.TransID
LEFT JOIN UserInfos ui ON ui.Username = t.WhoisMaking
LEFT JOIN Airports fr ON fr.AirPortName = t.PortofLading
LEFT JOIN Airports toLoc ON toLoc.AirPortName = t.PortofUnlading
ORDER BY t.TransID OFFSET :offset ROWS FETCH NEXT :fetchSize ROWS ONLY
```

Cần probe `INFORMATION_SCHEMA.COLUMNS` để confirm: `TransitETD`, `TransitETA`, `MAWBNo`, `OperationPIC`, `SalesmanName`, `ShipmentType`, `CustomerName`.

## Upsert + Naming

- Consumer dùng `transactionId` (unique) làm key. Get-then-update, null → insert.
- DTO: `TracingEvent`, Logic: `TransactionTrackingSyncLogic`

## Phases

1. **Probe schema**: tạo RPC tạm `probeHpsTables()` confirm column names
2. **Implement pipeline**: TracingEvent, SyncLogic, Producer, Consumer, Cron, Groovy, RPC
3. **Register HPS tenant** + Kafka topic config
4. **Test**: chạy `syncTransactionTrackings(HPS)` → verify tab UI

# D. Customs Receipt Bot v5

Project: customs-receipt-bot
Status: Approved

## D.1 Design — Modular + HTTP-first

### Giữ / Đổi / Out of scope

| Giữ | Đổi | Out of scope |
|---|---|---|
| Desktop CLI, PyInstaller | Selenium chỉ login → httpx cho API + download | Client-server, Web UI |
| Google Sheet logging | Feature-based codebase | Playwright, auto-captcha |
| Local storage theo ngày | pydantic, pytest, loguru, pymupdf thay pdfminer | |

### Cấu trúc thư mục

```
customs_receipt_bot/
├── pyproject.toml
├── src/customs_bot/
│   ├── __main__.py, cli.py, config.py, logging.py
│   ├── shared/ (models.py, http.py, paths.py)
│   ├── features/
│   │   ├── auth/ (selenium_login, session, account_pool)
│   │   ├── pdf_parsing/ (parser, extractors)
│   │   ├── receipt_fetch/ (api_client, downloader, pipeline)
│   │   ├── storage/ (local)
│   │   └── reporting/ (sheet_client)
│   └── build/
└── tests/ (conftest, e2e/)
```

**Nguyên tắc:** mỗi feature self-contained, cross-feature qua `shared/`, orchestrate qua `cli.py`.

### Flow chính

```
CLI → Auth.get_session() → [cookies hết hạn? → Selenium login (captcha tay)]
  → httpx.Client (cookies set)
  → loop mỗi PDF: Parse → API search → Download → Save → Log Google Sheet
```

Selenium chạy 1 lần/phiên → đóng Chrome ngay. httpx nhanh 5-10x.

### Library changes

| Hiện tại | Thay bằng | Lý do |
|---|---|---|
| `requests` | `httpx` | HTTP/2, pool |
| `pdfminer.six` | `pymupdf` (fitz) | ~10x nhanh, table extraction |
| `selenium 4.1` | `selenium 4.25+` | Upgrade |
| `print` | `loguru` | Structured log |
| (none) | `pydantic v2` | Validate config + models |
| 2 file test | `pytest` + `respx` | Test framework |
| `requirements.txt` | `pyproject.toml` | Modern packaging |
| `pyodbc` | **xóa** | Không dùng |

### Testing strategy

- Unit tests mỗi feature, `pdf_parsing`: PDF samples anonymized
- `receipt_fetch/api_client`: mock httpx bằng `respx`
- **Không test** `selenium_login` (smoke thủ công)
- Coverage: 70%+ non-selenium code

### Migration phases

| Phase | Nội dung | PR |
|---|---|---|
| 0 — Setup | pyproject.toml, pytest, ruff, loguru, pydantic, skeleton | 1 |
| 1 — Bóc tách | Move legacy → features/ (no behavior change) | 1 |
| 2 — Core change | httpx thay Selenium navigation (hoặc 2b: giữ Selenium, wrap sạch) | 1 |
| 3 — pymupdf | Rewrite parser, snapshot tests | 1 |
| 4 — Cleanup | Xóa code cũ, update build | 1 |

### Risks

| Risk | Impact | Mitigation |
|---|---|---|
| Cookies Selenium → httpx fail | Cao | Spike test Phase 2: copy UA + headers |
| ASP.NET `__VIEWSTATE` | TB | Inspect DevTools, fetch viewstate nếu cần |
| `pymupdf` output khác → regex fail | TB | Snapshot tests trước khi swap |
| PyInstaller bundle `pymupdf` | Thấp-TB | Test build sớm Phase 0 |
| Refactor vỡ production | Cao | Entry point cũ chạy song song đến Phase 4 |
